各授業回の学習事項

第1回 WWW と HTML

今回の授業では、JavaScriptを使ったイベント処理とDOM操作を学びました。HTMLの入力フォームにJavaScriptを組み合わせることで、ユーザーの操作に応じてメッセージを表示するなど、Webページを動的に変化させる方法が理解できました。初めはボタンをクリックしても反応がなかった状態から、イベントハンドラを設定することでクリックイベントに反応し、「こんにちは」といったメッセージを表示できるようになりました。また、querySelectorを用いてHTML要素を取得し、その内容を読み書きできることも習得しました。

演習の成果: 演習1-4:HTMLのさまざまな要素

第2回 CSS による Web ページの装飾

今回の授業では、JavaScriptを使ったイベント処理やDOM操作を学び、Webページを動的に操作する方法が身につきました。HTMLで作成した入力フォームにJavaScriptを組み合わせることで、ユーザーの入力に応じたメッセージ表示など、インタラクティブな機能を実装できることがわかりました。 最初はボタンをクリックしても何も起こりませんでしたが、イベントハンドラを登録することでクリックに反応し、「こんにちは」といったメッセージを表示できるようになりました。また、querySelectorを使ってHTML要素を取得し、その内容を取得・変更する方法も学びました。

演習の成果: 演習2-3:ウェブブラウザ説明ページの装飾

第3回 JavaScript の基本

今回の授業では、JavaScript を使ったイベント処理と DOM 操作について学びました。HTML で作成した入力フォームに対して、JavaScript のイベントハンドラを登録することで、ボタンのクリックに反応して「こんにちは」などのメッセージを表示できるようになりました。また、querySelector を使って HTML 要素を取得し、その内容を取得・変更する方法も理解しました。このように、Web ページを動的に操作する技術は、プログラミングスキルとしてだけでなく、将来的に社会人として働く際にも役立つと感じました。

演習の成果: 演習3-6:オブジェクトの参照

第4回 イベント処理とDOM(1)

今回の授業では、JavaScript を使ったイベント処理や DOM を通じて、Web ページを動的に操作する方法を学びました。HTML で作成した入力フォームに JavaScript の処理を加えることで、ユーザーの入力に応じたメッセージの表示など、インタラクティブな機能を実装できることがわかりました。最初はボタンをクリックしても何も起こらない状態でしたが、イベントハンドラを登録することで、クリックに反応して「こんにちは」などのメッセージを表示できるようになりました。また、querySelector などを使って HTML 要素を取得し、その内容を取得・変更できることも学びました。このような知識は、単なるプログラミングスキルにとどまらず、将来的に社会人として働く際にも役立つと感じました。

演習の成果: 演習4-10:足し算ページ

第5回 DOM(2)

今回の授業では、JavaScriptを使ってDOMを操作する方法を学びました。前回の基本的な要素検索や値の取得に加えて、今回は要素の新規作成、編集、削除など、より自由にWebページを動的に変更する技術に触れました。具体的にはcreateElement()で新しい要素を作成し、insertAdjacentElement()で特定の場所に挿入する方法を体験しました。例えば、リストの「サンダル」の後ろに「ヨット」を追加したり、画像要素にsrc属性を追加してブルームーンの画像を表示したり、段落の後ろに拓殖大学のリンクを設置したりしました。また、remove()メソッドを使うことで、特定の要素を削除することもできることを学びました。さらに、複数の要素をまとめて作成し、ページに追加する方法も練習しました。これにより、複雑なHTML構造をプログラムで生成する力がつきました。この内容をまとめて関数として外部JavaScriptファイルに記述し、ボタンのクリックイベントに紐づけることで、実際にページ上で動作させることができました。こうした操作を身につけることで、Webページを自由自在に操作・編集できるスキルがぐっと向上しました。今後は、今回のDOM操作の知識を使い、実際のプロジェクトで役立つ動的なWebコンテンツ作成に挑戦していきたいと思います。

演習の成果: 演習5-7:DOMプログラミング

工夫点，苦心点，強調点，感想

工夫点

この「Webコンテンツ作成の成果」において、デザインに統一性を保つことを重視しました。HTMLとCSSの授業で学んだ知識を活かし、全体の見た目やレイアウトを統一するために、フォントの選定やカラーリングに一貫性を持たせることで、ユーザーが違和感なくコンテンツを閲覧できるように工夫しました。

苦心点

JavaScriptはJavaに似ている部分もありましたが、その違いを理解するのに苦労しました。特に、DOM操作やイベント処理に関する部分では、何度もエラーが発生し、その都度デバッグに時間を取られました。結果として、エラーの原因を特定し、修正するための スキルを大幅に向上させることができました。 授業が進む中で、複数の課題を同時に進めなければならない場面が多くありました。特に、プロジェクト課題の進捗を管理するのが難しく、一つの課題に集中して他の課題が疎かになってしまうこともありました。

強調点

プロジェクトを進めるにあたり、授業の内容をしっかりと理解しようと心がけ、講義資料をよく確認しながら取り組みました。それでもわからない部分があった場合には、SEの方や教授に積極的に質問して理解を深めるよう努めました。 また、自分が理解できている部分については周囲の友達に説明するなど、互いに助け合いながら課題に取り組むことができました。その結果、最終的に課題を完了させることができました。 授業で学んだ内容に加え、HTMLとCSSについても自主的に学習を進めたことで、最終的に課題を期限内に提出できたと感じています。

感想

このプロジェクト課題を通して、Webコンテンツ作成に関するスキルをさらに深めることができました。これまでにHTML・CSS・JavaScriptを使ったWebページの制作に取り組んだ経験はありましたが、今回の課題ではより実践的かつ応用的な内容が多く、授業で学んだ知識を総合的に活用することで、より一層のスキル向上につながりました。 また、プロジェクトを進める中で直面したさまざまな課題や困難を乗り越えることで、問題解決力や他者とのコミュニケーションの重要性を改めて実感しました。こうした経験は、今後の学習や将来の仕事においても大きな財産になると感じています。 プロジェクトをやり遂げたことで得られた達成感は非常に大きく、自分の成長を強く実感する機会となりました。今後もWeb開発のスキルを継続的に磨き、多くの人に役立つコンテンツを提供できるよう努力していきたいと思います。また、新しい技術にも積極的に挑戦し、常に最新のトレンドを意識しながら自己研鑽に励んでいきたいと考えています。
